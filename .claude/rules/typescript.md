# Rules for src/ TypeScript files

These rules apply when editing files under `src/`.

## Build

- TypeScript config: `tsconfig.json` ‚Äî `strict: true`, `target: ES2022`, `module: Node16`.
- Build: `npm run build` (runs `tsc`). This is the type-check gate.
- **`npm run build` must pass before any PR.** Vitest does not type-check (G8).

## Conventions

- Match the style of neighboring files in the same directory. Check 2-3 before writing.
- `strict: true` is on ‚Äî no `any` without justification, no implicit `undefined`.
- ESM project (`"type": "module"` in package.json). Use `import`/`export`, not `require`.
- Module resolution is `Node16` ‚Äî use full file extensions in relative imports if required
  by neighboring code.

## Key files ‚Äî touch with care

- `src/automerge.ts` ‚Äî the merge gate. Deterministic by design (rule 5). Do not add LLM
  calls here. The L1‚ÜíL2‚ÜíL3 trust ladder is enforced here (issue #32).
- `src/protocol/labels.ts` ‚Äî label conventions (`agent:X`, `status:X`, `trust:L1/L2/L3`).
  Changing label semantics requires a SPEC change.
- `src/dispatch/adapter.ts` ‚Äî the `FighterAdapter` interface all adapters implement.
  Adding a field here means updating every adapter.

## What not to touch

- `dist/` ‚Äî build output, regenerated by `npm run build`.
- `loop-budget.md`, `loop-run-log.md` ‚Äî runtime loop state.
